Sxript Programming Language

Signature of a Medieval Illiterate
- Frederick W. Umminger Jr., Yale Record, 1959

Introduction:

Sxript is a high-level, interpreted, functional/procedural general-use programming language designed for numerical evaluation and problem solving with vectors, matrices, calculus, linear algebra, differential equations, and complex data types. The Sxript language utilizes variable storage, recursive functions, sub-programs with private scope, high-order functions, lambda calculus, plus other elements such as file IO and graphical output. The mathematics subsystem, formerly known as the STxAxTIC Math Library, is designed for arbitrary-precision calculations.

Sxript may be used as a stand-alone tool, or one may #include the Sxript apparatus as part of another program. Prototyped in a BASIC dialect, the main codebase is QB45 compatible; designed for use with modern QB64 and FreeBasic compilers. Implementations in C-family languages, namely C++ and JavaScript, are a near one-to-one translation of the prototype codebase. Sxript is a Rosetta Stone in this sense, expressing the same instructions in several languages independently.

Scientific Notation
Numbers significantly large (or sufficiently small) are displayed in scientific notation. The symbol that represents positive "power of ten" is *10^N, and for negative exponents, /powi(10,N). The notation *10^(-N) is also allowed, but isn't as fast as powi(). Scientific notation can be toggled with the sci( ) operator. An argument of 0 disables scientific notation, and an argument of 1 enables it. To check that scientific notation is enabled or not, use the command sci?() (with no argument).

Accuracy Factor
The global constant that dictates the precision of numerical output can be adjusted on-the-fly with with operator acc(x), where x is the number of numerical characters retained in a number. For a high-precision test, we first choose a high number, for instance, 398. Next we input a problem that will surely occupy all 398 digits, such as a repeating decimal. Choosing the square root of 2 for an example, we type sqr(2). The result is precisely 5 lines long, as the console is 80 characters wide. (The sign and the decimal take up the two extra spaces.)

Order of Operations
The "order of operations" obeyed by Sxript matches the standards of science and mathematics. (We will not imitate the erroneous behavior of certain systems such as spreadsheet programs.) Specifically, structures in parentheses are interpreted first, and in the order from "most embedded" to "least embedded". Next, any calculations involving the exponent ( ^ ) operator are interpreted into pow_( ) functions. As expected, all multiplication and division operations are performed, followed by addition and subtraction. Finally, the logical AND and OR operators, & and |, respectvely, are evaluated.


Variable Types
     (return to contents)

Real Numbers
In Sxript, there is only one type of real number - there is no distinction between single precision, double precision, floating point, long, etc. We have one number type here with arbitrary* decimal precision (* some operations return double precision only). Read more about the numerical accuracy factor above. All numbers are formatted with precisely one sign and one decimal point. For numbers outside the neighborhood of zero, scientific notation is used.

Numeric literals can be expressed simply by writing a number. The format of the number need not be strict at the user input level, meaning cos(3) is just as good as cos(+3.0). When a correctly-formatted numeric literal is sent to the interpreter, the number slips through with no new changes, signaling that evaluation is finished.

The num(a,y) operator is used for storing a number. The letter a is set to hold whatever y might stand for, so long as it boils down to a real number. Representing a number by its name is done with the "pound" operator #( ). For instance, to multiply a by +3.0, one would type #(a)*3 or, equivalently, 3*#(a).

String Variables
Strings, like numbers, can be stored, manipulated, and be processed through functions. Strings literals should be wrapped between the "un-shifted tilde" ( ` ) symbol and the the "single quote" ( ' ) symbol, and when the interpreter detects a raw string as input, it slips through untouched. (In fact, comments in the Sxript language are led by this same symbol.)

A string variable is stored by using the string(a,`y') operator, where a is the name of the string, and y is the contents of the string. Note that an unwrapped string will still pass through this function, but any blank spaces are closed. A string is represented through its variable name by the "string" operator $( ), which takes a string name as an argument. The sole job of $( ) is to swap the string name for its literal representation.

Vectors and Lists
The term "vector" shall be synonymous with the term "list", which is defined as a collection of items (numbers or strings, other vectors, whole matrics, and so on). Each item (or argument or element) is separated by the semicolon ( ; ) symbol, and the whole vector is contained in sharp brackets, ( < > ). The number of items or elements in the vector/list is called its dimension. Vectors of equal dimension can be added, in where elements of similar index combine to make a new vector of the same dimension. Vectors can also be "stacked", which appends one vector at the end of another, resulting in a vew vector of combined dimension. A vector can contain another vector, or even a whole matrix in any element.

A vector can be created and/or stored by the vect(a,y) operator, where a is the name of the vector, and y is the representation of the vector. A vector can be represented literally, for example <1;2;3;4>, or symbolically by the v( ) operator, which takes a vector name as an argument. The sole job of v( ) is to swap the vector name for its literal representation.

Vectors are, generally speaking, something I call "weakly evaluated". For instance, the vadd(a,b) merely inserts a plus ( + ) symbol between the respective elements. The resultant vector is not evaluated. To finish the job, use the veval( ) operator.

Matrix Nomenclature
A matrix looks like a block of numbers. More specifically, each row a matrix constistutes a row vector, and each each column makes a colummn vector. The dimensions of a matrix are specified by two numbers, denoted rows (counting down) and columns counting across. For instance, a square matrix with three entries on a side will have dimension (3,3). Add an extra row to the matrix, and the size becomes (4,3).

Two matrices can be added if they are of equal dimension, or multiplied when their dimensions obey (n,m)x(m,p), resulting in a matrix of dimension (n,p). There are no such things as matrix subtraction or division. A matrix, much like a function, can "operate" on a vector, transforming the vector into a new one. In this case, the number of colums in the matrix must match the number of rows of the vector.

The rows (vectors) of a matrix are separated by the vertical slash symbol "|" ( shift + \ ), and the entire matrix is contained in "square" brackets [ ].

Sub-Program Scope
The Sxript language supports local and global varibles. When control passes to the inside of a run( ) structure (or to a text file program), all previously-defined variables are available to read, and also temporarily rewrite. There is no need to pass arguments to a sub-program. When control exits the run( ) structure (or text program), all numbers and strings restore to their previous values. Only the information passed by print_ escapes the run( ) structure.

Two keywords have been designated to control the scope of a variable: global( ) and local( ). All variables default to local. When explicitly set to global, a variable "survives" the reset process when leaving a sub-program. The following program (output included) demonstrates the concept:

String Comparison
Beyond the simple check for exact equality, comparison between strings is ambiguously defined in the general case. We shall proceed with the "rule" that strings are composed of individual ASCII characters from left to right. Non-Embedded Special Functions
There is a subset of functions that must be called on their own. These are called "non-embedded" functions.

acc(x) (set digit precision)
Sets global constant STxAxTICAccuracyFactor to x, which must be an integer.

acc? (global precision check)
Returns the present value of the global constant STxAxTICAccuracyFactor.

accd(x) (set double precision)
Sets global constant DoubleAccuracyFactor to x, which must be an integer. CAUTION!

accd? (double precision check)
Returns the present value of the global constant DoubleAccuracyFactor.

appendn(name, record) (add record to file)
Opens the file specified by name and adds the string record to the bottom of the contents after being sent through EvalStructure(). The file is then saved and closed.

int< f([x]) ; lowlimit ; highlimit ; step > (one-dimensional integration)
Integrates the function f([x]) on the interval [lowlimit : highlimit] with a discete "dx" equal to step.

plot< f([x]) ; lowlimit ; highlimit ; step > (plot f(x) SCREEN 12)
Plots the function f(x) on the interval [lowlimit : highlimit] with a discete "dx" equal to step. Requires SCREEN 12.

plotascii< f([x]) ; lowlimit ; highlimit ; step ; recipient > (plot f(x) ASCII)
Plots the function f(x) on the interval [lowlimit : highlimit] with a discete "dx" equal to step. The output is represented as ASCII characters. For output to a local interface, let recipient be the "at" symbol: @. For chatroom interface, replace "@" by a pound sign followed by the name of the channel. For instance, #qb64.

sci(x) (toggle scientific notation)
Toggles scientific notation. The argument x is irrelevant. Use this function as you would an on-off switch.

sci?() (report scientific notation toggle)
Returns the status of the scientific notation (enabled or disabled) in the form of a string. The argument passed to this function is irrelevant.

exe('file) (execute program)
Invokes the Sxript interpreter to evaluate the contents of file.


Numerics

Math Operations

+ (add)
- (subtract)
* (multiply)
/ (divide)

     Precision: HIGH
The add, subtract, multiply, and divide operations each support high-precision math. Decimals and negative numbers are also supported.

!(x) (factorial)

     Precision: CONDITIONAL
The factorial operator !(x) calculates the product x*(x-1)*(x-2)*...*1. For high precision-calculations, the input argument x is limited to integers only. For non-integer arguments, an approximation up to SINGLE precision is returned.

\(x,y) (integer division)

     Precision: HIGH
The integer division operator \(x,y) computes the "normal" answer to x/y, and then truncates anything after the decimal point, without rounding.

abs(x) (absolute value)

     Precision: HIGH
Returns the absolute value (with a plus sign) of the argument.

choose(x,y) (combinatorial choose operator)

     Precision: CONDITIONAL
The choose operator, also known as the "binomial coefficient", is used extensively in combinatorics and probability theory. The quantity "A choose B" is written choose(A,B), which is defined as the ratio A!/((A-B)!B!). For integer arguments, high precision is allowed. For non-integer arguments, an approximation up to SINGLE precision is returned.

dice(x) (random integer)

     Precision: DOUBLE
Returns a random integer between +1.0 and the argument x, inclusive.

dom(x,y) (return larger number)

     Precision: HIGH
Returns the larger of two arguments, x or y.

ee(x) (digits of e)

     Precision: HIGH
Returns Euler's constant (+2.71828...) to precision x.

exp(x) (Euler exponent operator)

     Precision: CONDITIONAL
Returns Euler's exponential constant raised to the argument x, namely +2.71828...^x.

inrange(a,x,b) (if in range)

     Precision: HIGH
Returns +1.0 if x is greater than a and less than b. Returns +0.0 otherwise.

intg(a) (integer operator)

     Precision: HIGH
Returns the rounded-down integer approximation to a number.

kron(x,y) (Kroneker delta)

     Precision: HIGH
Returns +1.0 if the input arguments are equal, and returns +1.0 otherwise.

ln(x) (natural logarithm operator)

     Precision: HIGH
The natural logarithm operator ln(x) computes (x)^y essentialy "undoes" the Euler exponential operator. This function returns the natural logarithm of its argument x.

log(x) (base-10 logarithm operator)

     Precision: DOUBLE
The base-10 logarithm operator log(x) computes (x)^y essentialy "undoes" the power-of-ten exponential operator. This function returns the base-10 logarithm of its argument x.

mod(x,y) (modulus operator)

     Precision: HIGH
Returns x MOD y.

phi(x) (golden ratio)

     Precision: HIGH
Returns the golden ratio (+1.61803...) to precision x.

pi(x) (digits of pi)

     Precision: HIGH
Returns digits of pi (+3.14159...) to precision x.

pow(x,y) or x^(y) (decimal exponent operator)

     Precision: DOUBLE
The decimal exponent operator ^() computes (x)^(y), where the "up" argument must be enclosed in parentheses. This mode supports decimal arguments, for instsance x^(2.5). The maximum precision of this operator, however, is DOUBLE. A synonym for this operator is pow(x,y). The decimal exponent operator ^() is all one needs for "everyday" math.

powi(x,y) (integer exponent operator)

     High Precision (Fast): YES
The integer exponent operator powi(x,y) is a "fast" function that computes (x)^y, but only works when both arguments are integers. This operator lends itself naturally to scientific notation, and is employed for that purpose in the STxAxTIC Library. This is also idea for many "largenum" applications.

powl(x,y) (large exponent operator)

     Precision: HIGH
The large exponent operator powl(x,y) computes (x)^y, where the "up" argument must NOT be enclosed in parentheses, and must be an integer. The "down" argument x is allowed to any decimal or negative number.

prime(x) (prime number check)

     Precision: CONDITIONAL
Returns +1.0 if x is a prime number, and +0.0 otherwise. CAUTION: Slows down for very large numbers.

rnd(x) (random number)

     Precision: DOUBLE
Returns a random number between +0.0 and the argument x.

smaller(x,y) (if smaller)

     Precision: DOUBLE
Returns +1.0 if the first argument x evaluates to a number numerically less than y. Returns +0.0 otherwise.

sqr(x) (square root) (x >= 0)

     Precision: HIGH
The square root function is called with the operator sqr(x), where x is the square to root. For arguments that stay entirely within DOUBLE accuracy, the host's SQR( ) function is used to compute the result. When more accuracy is needed and the argument is an integer less than 19, a VERY fast technique is used to compute the result, called the "Babylonian method". For large-output square roots, a brute-force method is used to quickly compute the answer, but the input length is limited to DOUBLE precision. For all other square root requests, a first-order Newton's method is used. Prototyped in the library are several other square root methods, including a second-order approximation. These are presently not in use.

num(a,y) (variable store)

     Precision: HIGH
The var operator num(a,y) is a tremendously useful tool. This function evaluates (through EvalStructure) the expression contained in y, and is stored in name a.

numn(a,y) (evaluated variable store)

     Precision: HIGH
Counterpart to num(a,y), the numn(a,y) operator sends the argument y through EvalInput instead of EvalStructure before storage in a.

Vector Operations

v(a) (vector name -> literal operator)
Converts a vector stored under name a into its literal representation, i.e. <1;2;3>.

vect(a,y) (create vector manually)
Stores the vector y under the name a. The vector y may be a literal expression, as in <1;2;3> or a symbolic representation, as in v(foo).

vecn(bar,N) (create vector automatically)
Creates and stores a vector with of N. By default, vectors creted with vecn( ) have components consiting of integetrs counting upward from unity.

vadd(a,b) (lazy vector addition)
Inserts a plus ( + ) symbol between corresponding elements of two vectors. The vadd(a,b) operator requires that two vectors a and b are of equal dimension, and the returned result has dimension equal to that of a and b. The result is not evaluated.

vapply(foo,baz) (apply function to vector)
Applies the single-variable function foo to each element of the vector baz, and returns the new vector in evaluated form.

varg(foo) (vector argument count)
Returns the number of components/elements/items in vector foo, supposing it's defined.

vcadd(a) (vector collapse by addition)
Combines the components of a vector by addition, and returns the total.

vcmul(a) (vector collapse by multiplication)
Combines the components of a vector by multiplication, and returns the total.

vcross(a,b) (cross product)
The "cross product", or "vector product", multiplies two vectors and returns a new vector that is perpendicular to the original two. The vcross(a,b) operator requires that two vectors a and b have dimension 3.

vdot(a,b) (dot product)
The "dot product" multiplies corresponding elements in two vectors of equal dimension, and each of partial sums are combined into a single return scalar.

velem(foo,N) (vector component selection)
Returns the Nth component of a vector foo without evaluating.

velev(foo,N) (vector component evaluation)
Evaluates and returns the Nth component of a vector foo.

veval(foo) (vector component evaluation)
Evaluates and returns all components of a vector foo.

vins(foo,N,baz) (insert new element)
The vins(foo,N,baz) operator increases the dimension of foo by 1. Starting at poition N, all elements are shifted to the right by 1, and the element baz is placed at N.

vmap(cat,play,dog) (apply function of two vectors)
The vmap(cat,func,dog) requires each vector cat and dog to have the same dimension. The function play should be two variables {x} and {y}. The vmap function treats the elements in cat as the x-variable, and similarly, elements in dog correspond to y.

vpasc(N) (Pascal triangle)
Returns the Nth row of the Pascal triangle as a vector.

vrepl(foo,N,baz) (replace single element)
The vrepl(vec,N,new) operator replaces the element at position N in foo with element baz.

vshrink(foo,N) (delete one element)
The vshrink(foo,N) operator decreases the dimension of foo by 1. Starting at poition N+1, all elements are shifted to the left, thereby erasing the original element at N.

vstack(a,b) (stack two vectors)
Appends vector b at the end of vector a. The resultant vector has dimention equal to that of b and a combined.

vsub(vec,A,B) (return vector subset)
Returns a subset of the vector vec starting at position A, ending at position B, inclusive.

reduce(foo,baz) (reduce a list)
Uses a binary operation to combine elements in a list. The term foo must be the name a function (of up to two arguments), and baz is required to be a list/vector, as in <5;2;9;3;2>. For this example, the reduce(foo,<5;2;9;3;2>) is interpreted into:

foo(foo(foo(foo(5,2),9),3),2)

Note the parenthesis nest from left to right. To reverse the nesting, reverse the word "reduce". Indeed, the keyword ecuder(foo,baz) is interpreted as:

foo(5,foo(2,foo(9,foo(3,2))))
